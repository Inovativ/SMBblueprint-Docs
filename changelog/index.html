ChangeLog¶

This page contains an overview on all changes to the solution as new versions are released. Use the following command to update your current module version to the latest one:

update-module smbblueprint -scope currentuser

NOTE: Only the 2 latest version are actively supported. To ensure a stable deployment experience, it is advised to update to the latest release as soon as possible.

0.0.0.8 (10 February 2017)¶

Office 365¶

  • Office 365 Multi-License support
    • Both the GUI as CSV interfaces now allow the specification of multiple licenses to assign for new users
    • The CSV format is updated as followed:
      • The License header has become licenses
      • When multiple License SKU's are specified, they must be separated with a '|' character

        Example CSV:
        First,Last,Title,DisplayName,Department,Office,Mobile,Country,Groups,Licenses
        Jan,Van Meirvenne,Consultant,Jan.VanMeirvenne,ICT,Inovativ,32478707741,BE,TestGroup1,ADALLOM_O365|POWER_BI_STANDARD
        Jin,Van Meirvenne,Consultant,Jin.VanMeirvenne,ICT,Inovativ,32478707741,BE,TestGroup10,ADALLOM_O365|POWER_BI_STANDARD
        Jon,Van Meirvenne,Consultant,Jon.VanMeirvenne,ICT,Inovativ,32478707741,BE,TestGroup10,ADALLOM_O365|POWER_BI_STANDARD

Azure¶

  • Configurable Storage Tier for Azure Compute
    • In the GUI, a new dropdown allows you to choose the storage type to use for the virtual machines
    • In the CLI, a new parameter StorageType can be used to specify the desired storage Tier
    • Possible Values:
      • Standard_LRS (default)
      • Standard_ZRS
      • Standard_GRS
      • Standard_RAGRS
      • Premium_LRS
    • Check this resource for more info on the different storage types

General bugfixes and improvements¶

  • You can now define a custom password for the Office/Azure deployment from the GUI
    • A complexity check is added and will alert on a insecure password for both CLI/GUI operations
  • The solution now fully supports both CSP and non-CSP accounts
  • Azure / Office deployment options are hidden in the GUI if no appropriate subscription is available
  • When running a CLI or GUI command, an update-check will be performed and alert you if your module version is out-of-update
    • You can add the '-NoUpdateCheck' to any command to disable this behavior

0.0.0.7 (19 December 2016)¶

  • Multi-Region Support
    • Azure
      • The CLI Command 'New-SMBAzureDeployment' now has the additional parameters 'Location' and 'FallbackLocation'
        • The 'Location' parameter supports any Azure location name (eg westeurope). The resources will be deployed in the given location.
        • The 'FallbackLocation' parameter is used to specify where resources that are not supported in the chosen location should be deployed. This applies to the monitoring and automation resources, which are not widely available at the moment. The supported fallback locations are:
          • westeurope
          • southeastasia
          • australiasoutheast
      • The GUI follows the CLI changes by adding additional controls and prompts to generate the correct command for the deployment.
      • NOTE: For some locations, the backup feature is not yet supported. If one of these locations is chosen, the backup option will be disabled. The backup feature can not be deployed to the fallback location as it must exist in the same location as the VM resources.
    • Office
      • The GUI now allows you to specify any country when adding users
  • Server 2016 support
    • The 'New-SMBAzureDeployment' cmdlet now has a 'OS' parameter allowing you to deploy Server 2016 or 2012R2 VM resources
    • The GUI allows you to control this parameter through an added dropdown menu in the Azure options section.
  • General bugfixes and improvements
    • The login-screen now accepts the 'enter'-key to execute the logon procedure
    • Experimental changes to the UI to make it more compact
    • Fixed a possible deployment failure situation when the backup option was enabled and no additional VM was specified
    • Other internal or minor fixes
